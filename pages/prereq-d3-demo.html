Prerequisite map, D3-driven

Test page — not linked from Classes

Every course page's .prereq-map so far has been hand-typed HTML per course, and branches (an OR-group like EE2028's) use a CSS component whose curve/corner position is a hand-derived pixel formula — which turned out fragile: growing the corner's radius on the real EE2028 page made it visibly cut through its own station label, twice, before the formula got fixed. This page keeps prerequisite data in a plain JS object (see moduleData in this page's source) and has D3 compute real pixel coordinates for every station instead — the .prereq-station/.prereq-dot/ .prereq-label markers are the same real components (so city theme and dark mode still work for free), but the connector lines are a plain SVG drawn from those same coordinates, so a line and the station it touches can't drift apart. A branch's two lines converge on an invisible "ghost" node — a coordinate with no marker, just a routing waypoint — instead of a curve/corner formula. The map is also click-through: any grayed station is a real prerequisite of the one currently shown, click it to re-center the map on that module and see its chain.
